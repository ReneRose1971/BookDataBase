# Architektur-Grenzen

- Die Architektur-Dokumentation ist **deskriptiv**, nicht dogmatisch.
- Begriffe wie „Service“, „Controller“ oder „Repository“ sind **organisatorische Konventionen**, keine erzwungenen Architektur-Layer.
- Es dürfen **keine zusätzlichen Architektur-Layer** eingeführt werden, ohne expliziten Auftrag.
- **Keine Framework-Migrationen** oder Re-Architektur ohne expliziten Auftrag.
- **Technische Struktur** ist strikt von **fachlichen Regeln** getrennt.
- Dieses Dokument beschreibt **nur** den Ist-Zustand und ergänzt keine impliziten Konzepte.
